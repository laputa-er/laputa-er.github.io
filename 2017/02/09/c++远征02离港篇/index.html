3 c++ 语言 const 关键字

3.1 const

说明： 用来限制对值的写权限，常见用途包括

  • 提高代码的可读性
  • 防止函数对变量的修改

3.1.1 基本用法

const 与基本数据类型

说明：基本数据类型的变量使用 const ，使变量的值只读。

不使用 const 的内存模型

变量名|存储地址|存储内容
—|—|—
x|&x|3

使用 const 的内存模型

变量名|存储地址|存储内容
—|—|—
x|&x|3（只读）

const 与指针类型

通过p指向内容只读

变量名|存储地址|存储内容
—|—|—
x|&x|3
p|&p|&x

指针 p 只读

变量名|存储地址|存储内容
—|—|—
x|&x|3
p|&p|&x(只读)

指针 p 和 通过 p 指向内容都只读

变量名|存储地址|存储内容
—|—|—
x|&x|3(只读)
p|&p|&x(只读)

const 与引用(别名)

说明： 声明别名时如果被 const 修饰，则通过别名对变量的值的访问是只读的，但不影响其它方式对该变量的访问。

变量名|存储地址|存储内容
—|—|—
x|&x|3

  • 针对已有变量，声明指针或引用时，读写权限不能高于原变量。

4 c++ 函数新特性

4.1 函数特性

4.1.1 函数参数默认值

注意

  • 默认值只能在函数声明中提供（函数定义中给默认值在有的编译器中会出问题）
  • 默认值可以有一个或多个，但提供了默认值的参数，必须在参数列表的最右端
  • 没实参时则用默认值，否则实参覆盖默认值

4.1.2 函数重载

条件

  • 在相同作用域
  • 用同一函数名定义的多个函数
  • 参数个数或参数类型不同

4.1.3 内联函数

说明： 相比普通函数，內联函数没有真正的调用过程，因为编译时，函数调用语句就被函数体代码和实参代替了。
关键字： inline

理解內联函数

执行流：普通函数 vs 內联函数

  • 普通函数： 1->2->3->4->5
  • 內联函数：1->3->5（相比普通函数，內联函数没有调用的开销，因此性能更好）

5.1.3 申请内存注意事项

  • 使用 new 申请内存，使用 delete 释放内存。
  • 申请内存需要判断是否成功，释放内存需要设置空指针。
  • new 和 delete 配套使用，不要和 c 语言的方式混搭。

内存释放后，相关变量设置为 NULL（防止不小心使用了 p 或对同一块地址重复释放。如果不设置为 NULL， p 仍然指向那块释放了的内存地址，访问会出现无法预料的情况）。
